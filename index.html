12 学习shell脚本

摘要

  • 自动管理系统
  • 类程序
  • 不需要编译
  • 启动服务

12.1 什么是shell脚本

shell：一个接口，与系统进行沟通
脚本：一个程序，用shell写的。1.用到了shell的语法 2.正则表达式 3.管道、数据流重定向等。
总结：shell脚本是一个用shell语言写的程序（program）。shell脚本是纯文本文件。

12.1.1 为什么要学习shell脚本

自动化管理一台主机

12.1.2 脚本的编写与执行

block
编写注意事项

  • 命令从上到下，从左到右分析与执行
  • 命令、参数、选项间的多个空格、空行、会被忽略，tab键的多个空格被视为空格
  • 读到enter，则会进行执行该行
  • 一行内容过多，使用’[Enter]’来扩展至下一行

  • 为注释

执行方式

直接命令

  • 绝对路径 ： /etc/some.sh
  • 相对路径 : ./some.sh
  • 变量【PATH】功能 ：将.sh文件放在特定的文件夹下，如~/bin目录下，某用户的家目录的bin文件夹下

以bash程序执行

bash some.sh 或者 sh some.sh
注意：centos系统会默认将用户home目录下的bin文件夹添加至环境变量内

直接命令的方式执行，需要脚本有rx即可读，可执行的全选；而以bash程序执行，需要仅需文件有r即可读权限即可

程序编写分成数段

  • 第一行： #!/bin/bash说明此段脚本使用的shell名称——程序被执行人时，加载bash的相关配置文件。
  • 程序内容说明：除第一个#!的#为环境需要外，其余均为注释
  • 执行结果告知：exit命令让程序中断，返回一个数值给系统（一个命令执行的结果可用$?得到）

12.1.3 shell书写习惯

功能、版本、作者及联系、版权、历史记录、特殊命令用绝对路径、环境预先声明

12.2.2 脚本的执行方式差异

对比source、sh script、./script

直接执行的方式（直接命令执行./script、bash执行sh script）

用此种方式执行脚本时，该脚本会使用一个新的bash环境执行脚本内的命令。脚本在子进程的bash内执行。脚本执行完毕后，子进程bash内的所有数据便被删除。

source script

脚本在父进程中执行。各项操作在原本的bash内生效。不注销系统而让某些写入~/.bashrc的设置生效时，需要使用source ./bashrc。

12.3 判断test

检测某些文件或者相关的属性，使用test。但是执行结果不会显示任何信息，通过$?或者&&、||显示。

[zzy@iZwz98lrrmbxqv2j74r0j2Z bin]$ test -e ss.md && echo "exist" || echo "not exist"
not exist

常用参数

  • e : 是否存在
  • f : 是否为文件
  • d : 是否为文件夹
  • r : 有读的权限是否
  • w : 是否有写的权限
  • x : 是否有执行的权限
  • z : 是否为空字符串（判断字符串）
  • ! : 不具有（如，test ! -x filename 不具有可执行权限）

实例： 

#!/bin/bash
#Program
#   check the file exist or not ,what auth it has
#History
#2021/4/18  zzy v1
PATH=/bin:/sbin:/usr/bin:/usr/sbin:/usr/local/bin:/usr/local/sbin:~/bin
export PATH
echo -e "this program accept a name and check the messgae"
#1 input the name
read -p "input the filename: " filename
test -z ${filename} && echo "please input filename" && exit -1
#2 check the filename exist or not
test ! -e ${filename} && echo "${filename} not exist" && exit -2
#3 check file or dic
test -f ${filename} && filetype="file"
test -d ${filename} && filetype="dic"
#4 check is r t x
test -r ${filename} && perm="readable"
test -w ${filename} && perm="${perm} writable"
test -x ${filename} && perm="${perm} executable"
echo "${filename} is a ${filetype},and it's ${perm}."
